# Flash Design Context

## Users

Casual content consumers who want a curated feed of articles, videos, and podcasts without complexity. They value a clean, fast experience that feels personal and approachable. They're not RSS power users — they want content to come to them with minimal friction.

## Brand Personality

**Fast, clean, sharp** — speed and precision-focused, no-nonsense. But with warmth: playful touches in micro-interactions, friendly copy, and a UI that feels like it was made for you.

## Aesthetic Direction

- **Visual tone**: Playful and warm, but built on a clean, sharp foundation. Think Notion's friendliness meets Raycast's snappiness.
- **References**: Notion (personality in details, warm empty states), Raycast (speed, keyboard-first feel, polished animations)
- **Anti-references**: Dense power-user RSS readers (Inoreader, NewsBlur), overly corporate SaaS dashboards
- **Theme**: Light and dark mode, adaptive via Apple UIKit color system. Both modes are first-class.

## Design System

- **Colors**: Apple UIKit adaptive tokens (`text-text`, `bg-fill-secondary`, `bg-material-thick`, etc.). Brand accent `#FF5C00` (orange) used at 5-20% opacity for glassmorphic accents, borders, and glows.
- **Typography**: SN Pro font family. Apple-style type scale from `caption` (10px) to `largeTitle` (26px).
- **Icons**: MingCute with `i-mgc-` prefix (e.g., `i-mgc-star-cute-re`). Regular (`-re`) and filled (`-fi`) variants.
- **Motion**: Prefer CSS transitions for simple states. Framer Motion (`m.*` from `motion/react` with LazyMotion) for complex animations. Spring presets: smooth (0.4s, bounce 0), snappy (0.4s, bounce 0.15), bouncy (0.4s, bounce 0.3).
- **Elevated surfaces**: Glassmorphic — heavy backdrop blur (`backdrop-blur-2xl`), multi-layer shadows with orange accent at low opacity, gradient borders.
- **Spacing**: Tailwind defaults + safe area insets for mobile. Border radius via `--radius: 0.5rem`.

## Design Principles

1. **Speed is a feature** — every interaction should feel instant. Prefer optimistic updates, skeleton loaders, and spring animations that convey responsiveness. Never block the user.
2. **Quiet until needed** — default to minimal, clean surfaces. Use color, motion, and elevation to draw attention only when something matters (unread counts, new content, errors).
3. **Native-feeling** — follow platform conventions. UIKit colors on desktop/web, NativewindCSS on mobile. The app should feel like it belongs on the user's device, not like a generic web app.
4. **Warm details** — personality lives in micro-interactions, empty states, and transitions. A subtle bounce on a button press, a friendly empty state illustration, a smooth drawer slide. These moments make the app feel human.
5. **Accessible by default** — WCAG 2.1 AA. Good contrast ratios, keyboard navigable, screen reader friendly. Respect `prefers-reduced-motion`. Design for inclusion without sacrificing aesthetics.

## Accessibility

- WCAG 2.1 AA compliance
- Respect `prefers-reduced-motion` (use `f-motion-reduce` class)
- Keyboard navigation support on all interactive elements
- Proper ARIA labels on icon-only buttons
- Sufficient contrast ratios using the UIKit adaptive color system
